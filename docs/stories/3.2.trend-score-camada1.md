## Story

**As a** usuário de inteligência de tendências,
**I want** um score que ranqueie tópicos em ascensão,
**so that** eu veja o que está crescendo de verdade (não ruído de clusters pequenos).

## Acceptance Criteria

1. Implementa `TrendScore = volume·log(growth) + λ·max(0,z)` conforme `docs/design/trend-score.md` (§3), com `volume=log(1+R)`, `growth=(R+α)/(P+α)`, `z=(R/w−μ)/(σ+ε)`.
2. Guardas aplicadas: suporte mínimo `n_min`, suavização `α`, tópico novo (`P=0`) sinalizado com badge "novo".
3. Parâmetros configuráveis com defaults documentados (`w=7, α=1, H=60, λ=1.0, n_min=5, ε=1e-6`); função **testada unitariamente**.
4. Produz ranking de "Tópicos em Ascensão" consumível pelo dashboard (`scores.parquet`).

## 📘 Guia de Desenvolvimento (para o integrante)

> Dificuldade 🔴 — **liderada pelo técnico**. Guia enxuto e focado em rigor; o pseudocódigo da spec (§6) é a referência exata.

### 🎯 O que esta story faz
Transforma cada série temporal num **número único** que diz "o quanto este assunto está em alta agora", combinando: cresceu? (growth), tem volume relevante? (volume), está surtando contra a própria história? (z). É o **coração** do produto e o **plano B** caso a LSTM não funcione — por isso é MUST.

### 🧠 Pontos de rigor (não pular)
- **Fórmula exata da spec (§3):** `volume·log(growth) + λ·max(0,z)`. Não improvisar variações.
- **`log(growth)` é elegante:** positivo quando cresce, negativo quando encolhe → penaliza queda **sem `if`**.
- **Guardas são o que separam sinal de ruído:** `n_min` (ignora clusters minúsculos), `α` (evita divisão por zero), badge "novo" (não inflar score de tópico recém-nascido).
- **Determinístico → testável:** esta é a função **mais testada** do projeto (AC3). Use os exemplos da spec como casos de teste.
- **Tudo parametrizado:** ler `w, α, H, λ, n_min, ε` da config, nunca hard-coded.

### 🪜 Passo a passo (resumido — seguir pseudocódigo da spec §6)
1. Para cada tópico, recortar a série e calcular `R` (janela recente `w`), `P` (janela anterior `w`).
2. `growth=(R+α)/(P+α)`, `volume=log(1+R)`, e `z` sobre o histórico `H` (excluindo a janela atual).
3. Aplicar guardas: se `R < n_min` → marcar "amostra insuficiente"; se `P=0` e idade `< w` → badge "novo".
4. `trend_score = volume·log(growth) + λ·max(0,z)`; ordenar para gerar `rank`.
5. Salvar `scores.parquet`. Escrever os testes.

### ✅ Como saber que funcionou
- `scores.parquet` tem `{topic_id, T, R, P, growth, volume, z, trend_score, is_new, support_ok, rank}`. *(AC4)*
- Testes unitários cobrindo crescimento, encolhimento, tópico novo e suporte mínimo passam. *(AC3)*
- O ranking faz sentido numa inspeção rápida (assuntos quentes no topo).

### ⚠️ Erros comuns
- **Divisão por zero** → faltou `α`/`ε`.
- **Janela errada** → conferir índices `R=[T-w+1, T]`, `P=[T-2w+1, T-w]` (spec §3).
- **Score inflado de tópico novo** → faltou a guarda do badge "novo".

### 📚 Glossário (novos termos)
| Termo | Significado simples |
|---|---|
| Crescimento (growth) | Quanto a janela recente cresceu vs. a anterior |
| z-score | Quão "fora do normal" está, vs. a própria história |
| Suavização (α) | Pequeno valor que evita divisão por zero |
| Suporte mínimo | Volume mínimo para o tópico ser considerado |
