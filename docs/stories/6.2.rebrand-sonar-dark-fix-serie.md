11. Se, após o recorte, não sobrar nenhuma linha com atividade (`count_weekly` todo zero ou série vazia para o tópico), mantém o fallback já existente `st.info("Sem dados temporais para exibir.")` (`app.py:290`), sem lançar exceção.
12. A transformação (converter `data`, filtrar/recortar o período morto) é extraída para uma **função pura e testável**, sem I/O e determinística com dados sintéticos — ver decisão de localização em Dev Notes.

### 🧠 Pontos de rigor (não pular)

Se o gráfico continuar usando `count`, ele fica achatado em zero mesmo depois do fix de tipo de data.
- **Não tocar o pipeline de scores:** `count`/`count_weekly` já existem no parquet (`src/scores/series.py`, Story 3.1) — o fix é **só na camada de exibição** do dashboard (`app.py`), convertendo tipos e escolhendo qual coluna plotar. Não recalcular nada.
- **Cuidado com nomes de arquivo:** já existe `src/scores/series.py` (pipeline, Story 3.1) e `tests/test_series.py` (testa esse módulo). A nova função de preparação da série **para o dashboard** é um módulo/arquivo de teste diferente — não sobrescrever nem confundir com esses.
- **Rebrand é só de UI:** não mexer em `pyproject.toml` (`name = "..."`), `README.md`, nome de pastas/pacotes ou títulos de stories já fechadas. Só os textos que aparecem na tela do Streamlit.

### 📚 Glossário
| Termo | Significado simples |
|---|---|
| `count` vs `count_weekly` | Contagem diária vs. semanal de artigos por tópico — no schema atual, a diária é quase sempre zero/muito esparsa; a semanal concentra o sinal real |
| Eixo categórico (Plotly) | Quando o Plotly não reconhece a coluna como data, ele trata cada valor distinto como uma categoria separada no eixo — gera um eixo com milhares de "colunas" em vez de uma linha de tempo contínua |
| Período morto | Trecho da série sem nenhuma atividade (`count_weekly == 0`), presente desde o início da série (2021) até o começo do surto real de interesse pelo tópico |
| Tema dark nativo do Streamlit | Configuração de cores via `.streamlit/config.toml` (`base = "dark"`) — sem CSS customizado, sem dependência nova |
