## 📘 Guia de Desenvolvimento (para o integrante)

> Dificuldade 🟡. Streamlit deixa "fazer um site" quase tão fácil quanto escrever um script. Glossários anteriores valem.

### 🎯 O que você vai construir
A "cara" do produto: uma página web simples que mostra, em ranking, os assuntos em alta — e, ao clicar num assunto, abre os detalhes (gráfico no tempo, palavras-chave e as notícias). Tudo lendo arquivos prontos (rápido, sem internet).

### 🧠 Conceitos que você precisa entender
- **Streamlit:** ferramenta que transforma um script Python numa página web. Você escreve `st.title(...)`, `st.dataframe(...)` e ele renderiza.
- **Cache (`@st.cache_data`):** "lembrar" o resultado de carregar os arquivos para não reler a cada clique → deixa o app rápido (NFR8).
- **Drill-down:** "clicar para aprofundar" — da lista geral para o detalhe de um item.
- **Artefatos pré-computados:** o dashboard **não calcula nada pesado**; ele só **lê** o que as fases anteriores já salvaram.

### 🪜 Passo a passo
1. **Carregue com cache** (Task 1): escreva `load_artifacts()` decorada com `@st.cache_data`, lendo os parquets/JSON de `dados/scores` e `dados/topics`. *Por quê:* o cache é o que garante a tela em < 5 s.
2. **Monte o ranking** (Task 2): ordene por `trend_score` e mostre `label` + `growth` numa tabela ou em cards (`st.dataframe` ou `st.metric`).
3. **Faça o drill-down** (Task 3): um `st.selectbox` para escolher o tópico; ao escolher, plote a série (`plotly`), liste os top termos e os artigos (título + link). *Por quê:* é o que dá profundidade e credibilidade.

### ✅ Como saber que funcionou
- `streamlit run src/dashboard/app.py` abre o painel sem erro. *(AC1)*
- A lista mostra os assuntos em alta com nome e crescimento. *(AC2)*
- Clicar num tópico abre série + termos + artigos. *(AC3)*
- A tela principal aparece rápido (< 5 s). *(AC1/NFR8)*
