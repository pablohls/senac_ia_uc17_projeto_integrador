## 📘 Guia de Desenvolvimento (para o integrante)

> Dificuldade 🟡. Glossários anteriores valem.

### 🎯 O que você vai construir
Um "mapa de conexões": cada bolinha é uma palavra-chave e as linhas mostram quais palavras costumam aparecer juntas. Ajuda a enxergar a estrutura das tendências (ex.: "IA" ligada a "modelo", "lançamento", "open source").

### 🧠 Conceitos que você precisa entender
- **Grafo:** estrutura de "bolinhas" (nós) ligadas por "linhas" (arestas). Aqui: nós = termos, arestas = "apareceram juntos".
- **Co-ocorrência:** quando dois termos aparecem no mesmo contexto (mesmo artigo/tópico). Quanto mais vezes juntos, mais "forte" a linha.
- **NetworkX:** biblioteca que cria e organiza grafos. **Plotly** desenha de forma interativa.
- **Legibilidade:** grafos com termos demais viram "bola de pelo" — a gente limita aos mais relevantes.

### 🪜 Passo a passo
1. **Conte pares** (Task 1): para cada artigo/tópico, gere os pares de termos que aparecem juntos e some as ocorrências. *Por quê:* o peso da aresta = quantas vezes juntos.
2. **Monte o grafo** (Task 2): `G = nx.Graph()`; adicione nós (termos) e arestas (pares) com `weight`. Use um layout (`nx.spring_layout`) para posicionar.
3. **Desenhe** (Task 3): converta nós/arestas em uma figura Plotly (scatter de pontos + linhas) e mostre com `st.plotly_chart`. Adicione um controle (`st.slider`) para limitar nº de termos. *Por quê:* manter legível sobre o dataset.

### ✅ Como saber que funcionou
- O grafo aparece no dashboard, interativo. *(AC1)*
- Dá para ler os termos principais sem virar uma "bola" ilegível. *(AC2)*

### ⚠️ Erros comuns & soluções
- **Grafo ilegível** → limite aos top-N termos / arestas mais fortes.
- **Lento** → pré-calcule o grafo (cache) em vez de recalcular a cada interação.

### 🆘 Quando me chamar
- Para escolher o critério de "co-ocorrência" (por artigo vs por tópico) se ficar confuso.
