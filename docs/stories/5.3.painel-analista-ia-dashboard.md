## Story

**As a** usuário do dashboard,
**I want** ver o **nome em linguagem natural** e o **"por que está subindo"** de cada tópico,
**so that** eu entenda a tendência na hora — em vez de ler termos c-TF-IDF crus.

## Acceptance Criteria

1. O dashboard **lê `dados/insight/briefings.parquet`** (A5) com `@st.cache_data`, junto aos demais artefatos.
2. **Flag de disponibilidade `tem_insight`** (espelha o padrão `tem_camada2`): presente e não-vazio ⇒ usa `label_llm`; ausente/vazio ⇒ **cai graciosamente** para o `label` c-TF-IDF (`topic_info`) e esconde o texto de "por que sobe".
3. **Cards e tabela de ranking** passam a exibir `label_llm` quando disponível; o **drill-down** do tópico ganha o `why_summary` (nova aba ou bloco "🧠 Análise").
4. **Nenhuma regressão:** com o A5 ausente, o dashboard funciona exatamente como hoje (carga < 5s, NFR8). Rodar `poetry run streamlit run src/dashboard/app.py` sem erros nos dois cenários (com e sem A5).
5. Teste do carregamento/junção (AppTest ou função de load isolada) cobrindo os dois caminhos (com e sem A5). `pytest` **verde**.

## 📘 Guia de Desenvolvimento (para o integrante)

> Dificuldade 🟢 — **inicial**. É a story mais "de vitrine": pouco código, muito impacto visual. Copie o jeito que o `app.py` já trata os alertas da Camada 2 (`tem_camada2`).

### 🎯 O que esta story faz
Faz o dashboard **mostrar** o trabalho do Analista IA (Story 5.2): no lugar de `iphone apple`, aparece *"Prime Day pressiona preços de iPhone"*, e ao abrir o tópico há um parágrafo explicando o porquê. Se o arquivo do Analista IA não existir, tudo volta ao normal — sem erro.

### 🧠 Pontos de rigor (não pular)
- **Espelhe `tem_camada2`:** o `app.py` já tem o padrão exato de "recurso opcional que pode faltar". Copie a lógica para `tem_insight`.
- **Fallback é obrigatório (AC2, AC4):** sem A5, use o label antigo. Nunca deixe o dashboard quebrar por falta do arquivo do LLM.
- **Não recalcule nada (NFR8):** o dashboard só **lê**. A carga tem que continuar < 5s.

### 🪜 Passo a passo (resumido)
1. Ler `briefings.parquet` no `load_artifacts()` (opcional).
2. `tem_insight = briefings is not None and not vazio`.
3. Usar `label_llm` nos cards/tabela quando `tem_insight`.
4. Mostrar `why_summary` no drill-down.
5. Testar com e sem o arquivo.

### ✅ Como saber que funcionou
- Com o A5 presente: cards e tabela mostram nomes legíveis; drill-down tem "🧠 Análise". *(AC3)*
- Removendo o A5: dashboard funciona igual ao de hoje, sem erro. *(AC2, AC4)*
- `pytest` verde. *(AC5)*

### ⚠️ Erros comuns
- **Assumir que o A5 sempre existe:** ele é opcional; trate como os `alerts`.
- **Quebrar o join:** `briefings` casa por `topic_id`; use `merge(how="left")`.
