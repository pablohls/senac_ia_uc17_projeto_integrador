## 📘 Guia de Desenvolvimento (para o integrante)

> Dificuldade 🟢 e **sem programar**. Aqui você vira "jornalista de dados" + "responsável por ética". Ótima story para quem prefere escrever e analisar.

### 🎯 O que você vai construir
Dois textos que fecham o projeto com chave de ouro para a banca: (1) a **história de uma tendência** que o sistema detectou — contada de forma clara, com gráfico; e (2) a **discussão ética** — por que coletar esses dados é legítimo e responsável. Mais o **README** que ensina qualquer pessoa a rodar o demo.

### 🧠 Conceitos que você precisa entender
- **Estudo de caso:** contar **uma** história concreta (não falar do sistema em geral). Ex.: "Em [data], o tópico X disparou; o SONAR detectou Y dias depois do início; eis o gráfico".
- **LGPD (Lei Geral de Proteção de Dados):** lei brasileira sobre dados pessoais. Como usamos **só notícias públicas e sem dados pessoais (PII)**, estamos do lado seguro — mas precisamos **explicar isso**.
- **PII (dados pessoais):** informação que identifica alguém (nome, CPF, e-mail). Nosso projeto **não coleta** isso.
- **Alarme falso (falso positivo):** quando o sistema aponta uma "tendência" que não era real. É um risco honesto a discutir (nenhum sistema é perfeito).

### 🪜 Passo a passo
1. **Conte o caso** (Task 1): use o resultado do backtest (Story 3.4). Escreva 2-3 parágrafos: qual assunto, quando subiu, como o sistema captou, e o "porquê" por trás (o que aconteceu no mundo real). Inclua 1-2 figuras do dashboard. *Por quê:* a banca adora ver o sistema aplicado a um caso concreto.
2. **Escreva a ética** (Task 2): explique que os dados são **públicos** (notícias), que respeitamos `robots.txt` e os termos dos sites, que **não há dados pessoais**, e discuta o risco de alarme falso e como mitigá-lo (ex.: o suporte mínimo `n_min`). *Por quê:* responsabilidade é critério de avaliação.
3. **Finalize o README** (Task 3): liste os 3 comandos de reprodução e uma breve descrição de cada fase. *Por quê:* garante que o projeto seja reproduzível por terceiros (e pela banca).

### ✅ Como saber que funcionou
- Existe um documento/slide de estudo de caso com narrativa + figura. *(AC1)*

### ⚠️ Erros comuns & soluções
- **Caso vago/genérico** → escolha **um** evento concreto e conte a história dele.
- **Ética "decorativa"** → conecte às escolhas reais do projeto (sem PII, rate-limit, suporte mínimo), não frases genéricas.
