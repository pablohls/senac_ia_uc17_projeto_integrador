## 📘 Guia de Desenvolvimento (para o integrante)

### 🎯 O que você vai construir
Agora que temos a **lista de endereços** (Story 1.2), vamos **visitar cada link e copiar o conteúdo** (título e corpo da notícia), montando a tabela final de notícias — o **"dataset congelado"**. "Congelado" porque, depois de pronto, ele não muda: o demo inteiro roda em cima dele, sem internet.

### 🧠 Conceitos que você precisa entender
- **Web scraping / extração:** programar o computador para "ler" uma página da web e tirar só o que interessa (o texto da notícia), ignorando menus, anúncios e rodapé. A biblioteca `trafilatura` faz o trabalho pesado.
- **`doc_id` (hash da URL):** um "código de barras" único e fixo para cada artigo, gerado a partir do link. **É a chave mais importante do projeto** — ela vai conectar este artigo às suas fases seguintes (embedding, tópico, série). Nunca pode mudar.
- **Tratamento de erro (try/except):** alguns links vão falhar (página fora do ar, formato estranho). Em vez de o programa todo parar, a gente "tenta; se der erro, anota e segue".
- **Rate-limit:** esperar ~1 segundo entre downloads, para não sobrecarregar o servidor do portal (é ética e respeito ao `robots.txt` — NFR4).

### 🪜 Passo a passo
1. **Carregue a lista** (Task 1): leia `dados/raw/urls.parquet`. *Por quê:* é o resultado da story anterior — não recolete URLs.
2. **Visite e extraia** (Task 2): para cada URL, use `trafilatura.fetch_url` + `trafilatura.extract` para pegar título e texto. Coloque um `time.sleep(1)` entre as visitas. *Por quê:* o sleep respeita o servidor; sem ele você pode ser bloqueado.
3. **Proteja o lote** (Task 2): coloque cada extração dentro de `try/except`. No `except`, registre no log (`logging`) e continue. *Por quê:* com milhares de artigos, alguns sempre falham — não pode derrubar tudo.
4. **Gere o `doc_id`** (Task 3): algo como `hashlib.sha1(url.encode()).hexdigest()[:16]`. *Por quê:* é o código único que atravessa todo o pipeline.
5. **Monte e limpe** (Task 3): junte tudo num DataFrame com as 7 colunas do schema A1; remova duplicatas e textos vazios.
6. **Salve e versione** (Task 4): grave `dados/raw/corpus.parquet` pelo `io.py` e confirme que o Git LFS está cuidando dele.

### ⚠️ Erros comuns & soluções
- **Muito lento:** normal — são milhares de páginas a 1/s. Rode em janela tranquila; dá para salvar progresso parcial.
- **Texto vindo "sujo" (menus/anúncios):** confira se está usando `trafilatura.extract` (que limpa) e não pegando o HTML cru.
- **`doc_id` mudando entre execuções:** você gerou o hash de algo que muda (ex.: data de coleta). Gere **só a partir da `url`**.
- **Lote parando no meio:** faltou o try/except em alguma parte; envolva toda a extração.

### 🆘 Quando me chamar
- Se a taxa de falha de extração for alta (> ~20%) — pode ser bloqueio ou mudança no portal.
- Para revisar o schema final antes de "congelar" (este artefato alimenta todo o resto).
