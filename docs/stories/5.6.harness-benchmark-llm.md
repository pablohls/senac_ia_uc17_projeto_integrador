## Acceptance Criteria

1. `scripts/benchmark_llm.py` executa o protocolo (`protocolo-empirico.md`): para cada modelo da shortlist × amostra congelada, gera as saídas das tarefas T1 (rótulo), T2 ("por que sobe") e T3 (RAG), **reaproveitando `src/common/llm.py`** (só troca `base_url`/`model`).
2. **Amostra congelada** em `amostra_benchmark.json` (≈12 tópicos estratificados + 8 perguntas RAG, sendo 3 "pegadinhas"), fixa para os 3 modelos.
3. Coleta automática de **latência, throughput e VRAM de pico** por tarefa/modelo em `resultados_brutos.parquet`.
4. Gera **planilha de avaliação cega** (saídas embaralhadas/anonimizadas A/B/C) para pontuação humana pela rubrica 1–5.
5. Produz `resultado-benchmark.md` (no diretório de pesquisa) com a tabela consolidada e a recomendação; se o vencedor **não** for `qwen2.5:14b`, registrar a troca sugerida de `insight.model` e anotar no ADR-002 (Change Log).
6. **Declara limitações** (amostra pequena → sinal qualitativo, não significância estatística). Script reproduzível; não quebra se um modelo estiver ausente (pula com log).

## 📘 Guia de Desenvolvimento (para o integrante)

> Dificuldade 🟡 — **intermediária**, e **opcional**. Se o prazo apertar, pule: o ADR já tem um default defensável. Se der tempo, esta story rende uma **narrativa de rigor** forte na banca.

### 🎯 O que esta story faz
Roda os 3 modelos candidatos sobre os **mesmos** tópicos e perguntas, guarda as respostas e mede tempo/VRAM. Depois, duas pessoas da equipe pontuam as respostas **sem saber qual modelo é qual** (avaliação cega). No fim, um relatório diz qual venceu e por quê.

### 🧠 Pontos de rigor (não pular)
- **Avaliação cega (AC4):** embaralhe as saídas antes de pontuar — senão o viés ("gosto do Qwen") contamina o resultado.
- **Mesma amostra, mesmo prompt (AC1, AC2):** só o **modelo** muda. Congele a amostra num arquivo.
- **Fidelidade é eliminatória:** modelo que alucina no RAG (pegadinhas) perde, mesmo se escreve bonito.
- **Reúso (AC1):** use o `llm.py` da 5.1 — não escreva um cliente novo.
- **Honestidade (AC6):** 12 tópicos dão *sinal*, não prova estatística. Diga isso no relatório.

### 🪜 Passo a passo (resumido — protocolo-empirico.md)
1. Congelar `amostra_benchmark.json`.
2. `ollama pull` dos 3 modelos.
3. Loop modelo×amostra×tarefa via `llm.py`; medir tempo/VRAM.
4. Gerar planilha cega; 2 avaliadores pontuam (rubrica 1–5).
5. Consolidar `resultado-benchmark.md` + recomendação.

### ✅ Como saber que funcionou
- `resultados_brutos.parquet` tem as saídas dos 3 modelos para toda a amostra. *(AC1, AC3)*
- A planilha cega não revela qual é qual. *(AC4)*
- `resultado-benchmark.md` conclui com um vencedor + limitações declaradas. *(AC5, AC6)*

### ⚠️ Erros comuns
- **Avaliar sem cegar:** invalida o resultado.
- **Comparar quantizações diferentes:** padronize Q4.
- **Tunar o prompt por modelo:** o prompt é o **mesmo** para todos.

### 📚 Glossário
| Termo | Significado simples |
|---|---|
| Avaliação cega | Pontuar sem saber qual modelo produziu a saída |
| Rubrica | Régua de notas (1–5) por critério |
| Throughput | Tokens por segundo |
| Eliminatório | Critério que reprova sozinho (aqui: alucinar) |
