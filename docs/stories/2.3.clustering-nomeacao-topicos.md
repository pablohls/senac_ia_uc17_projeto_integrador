## Story

**As a** cientista de dados do SONAR,
**I want** agrupar os artigos em tópicos e nomeá-los automaticamente,
**so that** cada cluster seja um assunto legível, não um número.

## Acceptance Criteria

1. Pipeline **UMAP → HDBSCAN** (via BERTopic) gera clusters; outliers (`-1`) tratados.
2. **c-TF-IDF** extrai os termos representativos de cada tópico (rótulo automático).
3. Parâmetros documentados; **≥ 70%** dos tópicos avaliados manualmente julgados coerentes (gate de qualidade — SM2).

### 🎯 O que esta story faz
Pega os "vetores de significado" (Story 2.2) e **agrupa automaticamente** as notícias por assunto, sem ninguém dizer quais assuntos existem. Depois dá um **nome** a cada grupo a partir das palavras mais características dele.

### 🧠 Pontos de rigor (não pular)
- **Sem taxonomia pré-definida:** o objetivo é o sistema **descobrir** os tópicos sozinho (não classificar em categorias fixas).
- **Use os embeddings prontos:** passe `embeddings=` para o BERTopic; não deixe ele gerar de novo.
- **Outliers (`-1`):** o HDBSCAN marca artigos "sem grupo" como `-1`. Isso é esperado; trate-os (não force a um cluster).
- **Parâmetros importam:** `min_topic_size`/`n_neighbors` mudam muito o resultado. Documente o que usou e o **porquê** (a banca valoriza).

### 🪜 Passo a passo (resumido)
1. Carregar embeddings + textos alinhados por `doc_id`.
2. `BERTopic(embedding_model=None, umap_model=..., hdbscan_model=...)` e `fit_transform(textos, embeddings=emb)`.
3. Extrair `get_topic_info()` e os termos por tópico (c-TF-IDF) → `topic_info.parquet`, `topic_terms.parquet`.
4. **Validar:** olhar ~20 tópicos e julgar se o rótulo faz sentido; anotar a % coerente (meta ≥ 70%).

### ✅ Como saber que funcionou
- Existem `topic_info.parquet` (1 linha/tópico, com `label`) e `topic_terms.parquet` (termos por tópico). *(AC2)*
- Outliers `-1` estão presentes e tratados. *(AC1)*
- ≥ 70% dos tópicos da amostra julgados coerentes, documentado. *(AC3 / SM2)*

### ⚠️ Erros comuns
- **Tudo virou outlier** → `min_cluster_size` alto demais ou poucos dados; ajuste e documente.
- **Tópicos genéricos demais** → revisar UMAP `n_neighbors` e remoção de stopwords PT no c-TF-IDF.

### 📚 Glossário (novos termos)
| Termo | Significado simples |
|---|---|
| Clustering | Agrupar itens parecidos sem rótulos prévios |
| UMAP | Reduz a dimensão dos vetores p/ facilitar o agrupamento |
| HDBSCAN | Algoritmo que forma os grupos (e marca outliers) |
| c-TF-IDF | Acha as palavras que mais caracterizam cada grupo |
| Outlier | Item que não se encaixa em nenhum grupo (`-1`) |
