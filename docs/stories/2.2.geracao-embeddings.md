## 📘 Guia de Desenvolvimento (para o integrante)

> Dificuldade 🔴 — **liderada pelo integrante técnico**. Guia enxuto, focado nos pontos de rigor.

### 🎯 O que esta story faz
Transforma cada notícia num **vetor de números** (centenas de dimensões) que representa o "significado" do texto. Notícias parecidas geram vetores próximos — é isso que permite agrupá-las depois.

### 🧠 Pontos de rigor (não pular)
- **Alinhamento é sagrado:** a linha `i` do `embeddings.npy` **tem que** corresponder à linha `i` do `corpus_clean.parquet`. Por isso salvamos `embeddings_index.parquet` (`doc_id → row_idx`) **no mesmo passo**. Desalinhar aqui contamina todo o resto (é o risco nº 3 do architect-checklist).
- **Não reembedar depois:** o BERTopic (Story 2.3) recebe **estes** embeddings prontos; não deixe ele recomputar (custo dobrado de GPU).
- **Batch + GPU:** use `batch_size` adequado à VRAM; `device="cuda"`. Se cair em CPU, registre — roda, mas lento (fallback MiniLM).
- **`float32`:** persista nesse dtype (metade da memória do float64, suficiente).

### 🪜 Passo a passo (resumido)
1. Ler `corpus_clean.parquet` preservando ordem.
2. `SentenceTransformer(model_name, device="cuda").encode(textos, batch_size=64)`.
3. `np.save("dados/processed/embeddings.npy", emb.astype("float32"))` **e** salvar o index `{doc_id, row_idx}` na mesma função.
4. Logar tempo total.

### ✅ Como saber que funcionou
- `embeddings.npy` tem shape `[N, D]` com `N == len(corpus_clean)`. *(AC2)*
- `embeddings_index.parquet` tem `N` linhas, `doc_id` único, `row_idx` de 0..N-1. *(alinhamento)*
- Log mostra GPU usada e tempo total. *(AC1, AC3)*

### ⚠️ Erros comuns
- **`N` do `.npy` ≠ linhas do parquet** → desalinhamento; refazer no mesmo passo.
- **Out of memory na GPU** → reduzir `batch_size` ou trocar para MiniLM.

### 📚 Glossário (novos termos)
| Termo | Significado simples |
|---|---|
| Embedding | Vetor de números que representa o significado de um texto |
| Espaço vetorial | "Mapa" onde textos parecidos ficam perto |
| Batch | Processar em lotes (mais eficiente na GPU) |
| VRAM | Memória da placa de vídeo |

## Dev Notes

> Contexto de `docs/architecture.md` (v1.0 → Contratos A2, Tech Stack).

- **Owner:** `src/pln/embed.py`. Função: `embed(corpus_clean, model) -> (embeddings.npy, embeddings_index)`.
- **Entrada:** `corpus_clean.parquet`. **Saídas:** `embeddings.npy` (`float32 [N,D]`) + `embeddings_index.parquet` (`{doc_id, row_idx}`).
- **Modelo:** `config.embedding_model` (default `paraphrase-multilingual-mpnet-base-v2`, D=768; fallback `paraphrase-multilingual-MiniLM-L12-v2`, D=384).
- **Coding Standards (críticos):** **nunca** persistir `.npy` sem o `embeddings_index` no mesmo passo; embeddings consumidos pela 2.3 **sem recomputar**.
- **NFR2:** GPU em batch; tempo registrado.

### Testing
- **Local:** validação manual (estocástico/GPU não é unit-test). Checagem programática simples: `assert emb.shape[0] == len(corpus_clean)` e unicidade do index — pode virar um teste leve `tests/test_embeddings_align.py` com um corpus pequeno simulado (mockando o encode).
