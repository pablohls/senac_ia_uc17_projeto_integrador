## Story

**As a** desenvolvedor das features de IA generativa (Analista IA e RAG),
**I want** um único cliente de LLM que fale com qualquer endpoint OpenAI-compatible via configuração,
**so that** eu itere o prompt contra a Claude API e rode o demo 100% local (Ollama) **trocando só uma linha do `config.yaml`** — sem reescrever código.

## Acceptance Criteria

1. **Cliente único** em `src/common/llm.py` expõe `chat(messages, *, temperature, max_tokens) -> str | None`, falando o protocolo **OpenAI-compatible** (biblioteca `openai`), com `base_url` e `model` vindos da config.
2. **Degradação graciosa (inegociável):** qualquer falha (timeout, conexão recusada, erro do servidor, resposta malformada) faz `chat()` **retornar `None` com log de aviso** — **nunca** propaga exceção ao chamador. Isso é o que permite o dashboard seguir com labels c-TF-IDF quando o LLM está fora.
3. **Config-driven, zero número mágico:** classe `InsightParams` (pydantic) em `src/common/config.py` + seção `insight:` em `config/config.yaml` com **todos** os parâmetros (`base_url`, `model`, `temperature_batch`, `temperature_chat`, `max_tokens`, `rag_top_k`, `top_artigos`), validados e integrados ao `Config` raiz — no mesmo padrão de `TrendScoreParams`.
4. **Prova provider-agnostic:** demonstrar (em teste e/ou no relatório) que a **mesma** função responde contra **Ollama local** (`http://localhost:11434/v1`) e contra um endpoint OpenAI-compatible remoto, mudando **apenas** `insight.base_url`/`insight.model`.
5. **Dependência:** `openai` adicionada ao `pyproject.toml` (leve, **sem CUDA**); `poetry install` continua funcionando no M1 e na VM.
6. **Testes:** `chat()` mockado (determinístico) para uso das stories seguintes; teste explícito do **caminho de falha** (endpoint indisponível → `None`, sem exceção). Suíte `pytest` permanece **verde** (89+).

## 📘 Guia de Desenvolvimento (para o integrante)

> Dificuldade 🟡 — **intermediária**. Não tem ML aqui: é escrever uma "peça de conexão" limpa e bem testada. Se você fizer bem, as próximas 5 stories ficam fáceis.

### 🎯 O que esta story faz
Cria **uma função só** — `chat()` — que conversa com um "cérebro de texto" (o LLM). O truque: ela não sabe (nem se importa) **onde** o cérebro roda. Pode ser a Claude na nuvem (enquanto você testa) ou o Qwen2.5 rodando na sua GPU (no dia da banca). Isso é decidido por **uma linha** no `config.yaml`. É o padrão "adaptador": o resto do projeto fala com `chat()`, e só a config decide o resto.

### 🧠 Pontos de rigor (não pular)
- **Falhar é OK — travar NÃO é (AC2):** se o LLM estiver desligado, `chat()` devolve `None` e a vida segue. **Nunca** deixe uma exceção subir para quem chamou. Esse é o mesmo princípio da Camada 2 (LSTM) da Fase 3.
- **Nenhum número mágico (AC3):** endpoint, modelo, temperatura — **tudo** no `config.yaml`, validado por pydantic. Olhe como `TrendScoreParams` faz e copie o padrão.
- **Nada de chave hardcoded:** para Ollama a "chave" é fictícia; para endpoints reais, leia de variável de ambiente. Nunca comite segredo.
- **OpenAI-compatible ≠ OpenAI:** usamos a *biblioteca* `openai`, mas apontando para o Ollama. É só o "idioma" da API; o modelo é local.

### 🪜 Passo a passo (resumido)
1. Escreva `InsightParams` no `config.py` e a seção `insight:` no `config.yaml`.
2. Escreva `chat()` em `src/common/llm.py` com `openai.OpenAI(base_url=config.insight.base_url, ...)`.
3. Ponha tudo num try/except → log + `return None`.
4. `poetry add openai`.
5. Teste: um caso de sucesso (mock) e um de falha (mock que levanta erro → espere `None`).

### ✅ Como saber que funcionou
- `from src.common.llm import chat` funciona e, com Ollama no ar, `chat([{ "role":"user","content":"diga olá"}], temperature=0, max_tokens=20)` devolve texto. *(AC1, AC4)*
- Desligue o Ollama e chame de novo: devolve `None`, com um log de aviso, **sem quebrar**. *(AC2)*
- Trocar `insight.base_url` para a Claude (e a `model`) faz a mesma função responder — sem tocar no código. *(AC4)*
- `pytest` verde. *(AC6)*

### ⚠️ Erros comuns
- **Deixar a exceção subir:** o `except` tem que capturar de verdade (inclusive timeout e erro de conexão) e retornar `None`.
- **Esquecer o `api_key`:** o cliente `openai` exige *algum* valor; para Ollama use um dummy.
- **Import pesado no topo:** importe `openai` no módulo, mas não force carregamento de torch/etc.; esta peça é leve.

### 📚 Glossário (novos termos)
| Termo | Significado simples |
|---|---|
| LLM | "Modelo de linguagem" — o cérebro que gera texto |
| OpenAI-compatible | Um "idioma" de API que vários servidores (inclusive Ollama) entendem |
| Endpoint / base_url | O endereço onde o LLM atende (ex.: `localhost:11434/v1`) |
| Provider-agnostic | Não depende de *qual* provedor; troca por config |
| Degradação graciosa | Quando algo falha, o sistema continua funcionando com menos |
