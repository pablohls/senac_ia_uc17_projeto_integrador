## Story

**As a** usuário do TrendRadar,
**I want** que cada tópico em ascensão ganhe um **nome em linguagem natural** e um parágrafo explicando **por que está subindo**,
**so that** eu entenda a tendência de imediato, sem precisar decifrar listas de termos c-TF-IDF (`iphone apple` → *"Prime Day pressiona preços de iPhone"*).

## Acceptance Criteria

1. **Estágio batch** em `src/insight/run.py` (espelhando `src/scores/run.py`): lê `doc_topics`, `topic_info`, `topic_terms` e `scores`; seleciona os **tópicos em ascensão** (`topic_id != -1`, `support_ok=True`, ordenados por `trend_score`) e monta, por tópico, o contexto com os **`top_artigos`** artigos mais representativos (título + trecho).
2. **Geração via LLM:** para cada tópico, chama `chat()` (Story 5.1) e obtém **(a)** um `label_llm` curto (≤ 8 palavras) e **(b)** um `why_summary` (2–4 frases). Prompts fixos em `src/insight/prompts.py` (rotulação + "por que sobe").
3. **Contrato A5:** grava `dados/insight/briefings.parquet` `{topic_id, label_llm, why_summary, model_name, generated_at}` **via `src/common/io.py`** (`salvar_parquet`).
4. **Reprodutibilidade:** usa `temperature=config.insight.temperature_batch` (0.0); registra `model_name` no A5 **e** no `run_manifest.json` (`atualizar_manifest(stage="insight", ...)`).
5. **Degradação graciosa:** se `chat()` devolve `None` para um tópico → registra fallback (mantém o `label` c-TF-IDF original e `why_summary` vazio) **com log** e segue; se o LLM estiver totalmente indisponível → o pipeline **termina sem quebrar** (A5 pode não ser gerado; nada estoura).
6. **Execução:** roda por `poetry run python -m src.insight.run` sobre os artefatos congelados; testes mockam `chat()` e validam o **schema A5** e o caminho de degradação. Suíte `pytest` **verde**.

## 📘 Guia de Desenvolvimento (para o integrante)

> Dificuldade 🟡 — **intermediária**. Você vai "costurar" dados que já existem, mandar para o `chat()` da 5.1 e salvar o resultado. O segredo está em **bons prompts** e em **não quebrar** quando o LLM falha.

### 🎯 O que esta story faz
Pega os tópicos que estão bombando (já ranqueados pela Fase 3) e, para cada um, pergunta ao LLM: *"olhando estes 5 artigos, dê um nome curto para o assunto e explique em 2-3 frases por que ele está crescendo."* O resultado vira uma tabelinha (`briefings.parquet`) que a Story 5.3 vai mostrar no dashboard. **Roda offline, uma vez** — não é ao vivo (isso é o RAG, story 5.5).

### 🧠 Pontos de rigor (não pular)
- **Espelhe o `scores/run.py`:** mesma cara de orquestrador (logs, ler → processar → `salvar_parquet` → `atualizar_manifest`). Não reinvente o estilo.
- **Contexto é tudo (grounding):** o LLM só deve falar do que está nos artigos. No prompt, deixe claro: *"baseie-se APENAS nos títulos/trechos abaixo"*. Isso reduz invenção.
- **`temperature=0`:** no batch queremos resultado estável e reprodutível — não criativo. Vem da config, não hardcode.
- **Degradação graciosa (AC5):** se o LLM cair no meio, você **não** perde tudo nem quebra o pipeline. Cada tópico tem seu fallback (o label velho c-TF-IDF).
- **Registre o modelo:** `model_name` no A5 e no manifesto — para a banca saber com qual modelo o briefing foi gerado.

### 🪜 Passo a passo (resumido — recommended-approach §5A)
1. Ler artefatos, filtrar tópicos em ascensão.
2. Por tópico: pegar `top_artigos` (título+trecho) → montar prompt.
3. `chat()` → `label_llm` + `why_summary`.
4. Montar `briefings.parquet` (+ `model_name`, `generated_at`) e salvar via `io.py`.
5. `atualizar_manifest`. Fallback quando `chat()` devolver `None`.

### ✅ Como saber que funcionou
- Existe `dados/insight/briefings.parquet` com uma linha por tópico em ascensão. *(AC1, AC3)*
- Os `label_llm` são nomes legíveis (não listas de termos); os `why_summary` batem com os artigos. *(AC2)*
- O manifesto tem o `insight_model`. *(AC4)*
- Se você desligar o Ollama e rodar, o comando **termina** (sem A5, ou com fallback), **sem traceback**. *(AC5)*
- `pytest` verde. *(AC6)*

### ⚠️ Erros comuns
- **Prompt que "puxa" invenção:** peça explicitamente para se ater aos artigos; peça saída curta.
- **Recriar artefatos da Fase 2/3:** esta story só **lê** deles; a única escrita é o A5.
- **Deixar exceção subir quando o LLM falha:** trate por tópico e globalmente.
- **`generated_at` com `datetime.now()` sem timezone:** siga o padrão de `io.py` (UTC ISO).

### 📚 Glossário (novos termos)
| Termo | Significado simples |
|---|---|
| Batch | Processa tudo de uma vez, offline (≠ ao vivo) |
| Briefing | O texto-resumo que o Analista IA gera |
| Grounding | Fazer o LLM responder só com base no material dado |
| Contrato A5 | O formato acordado do `briefings.parquet` |
| Fallback | Plano B quando algo falha (aqui: usar o label antigo) |
